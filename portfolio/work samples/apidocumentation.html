API DOCUMENTATION

La Poche API

A REST API for managing products, orders, inventory, marketplaces, finance, and users on the La Poche platform.

Every endpoint below lists: Title, Overview, HTTP Method, Endpoint, Request Parameters, a Sample Request, Response Parameters, the Success Response (200/201/202) with a sample, the standard HTTP Error codes (400/401/403/404/429/500), and any Additional Error notes specific to that endpoint. All requests and responses use JSON.

OVERVIEW

Overview

The La Poche API gives programmatic access to everything available in the La Poche dashboard: product catalog, orders, inventory, marketplace connections, invoices and settlements, reports, and user accounts. It's the same API the dashboard itself is built on.

Base URL

https://api.lapoche.com/v2

Request format

All requests and responses use application/json. Send a Content-Type: application/json header on any request with a body.

Versioning

The current version is v2, included in the base URL. Breaking changes are released under a new version; La Poche supports the previous major version for at least 12 months after a new one ships.

AUTHENTICATION

Authentication

The API authenticates with a bearer token tied to a user account and its role. Generate a key under Settings → API Keys in the dashboard — this requires Administrator access.

Request header

Authorization: Bearer lp_live_4f8a2e9c1b3d6f7a8e2c5b9d1f4a7c3e

Keys inherit the permissions of the role they were generated under — a key created by a Warehouse Executive can only call inventory endpoints, even if the request is otherwise valid.

WARNING

Treat API keys as credentials. Don't commit them to source control or expose them in client-side code. Rotate a key immediately if you suspect it's been exposed.

ERRORS & RATE LIMITS

Errors & rate limits

HTTP status codes

CodeStatusMeaning
200OKRequest succeeded
201CreatedResource was created
202AcceptedRequest accepted; processing happens asynchronously
400Bad RequestMalformed request — check required fields
401UnauthorizedMissing or invalid API key
403ForbiddenThe key's role doesn't permit this action
404Not FoundResource doesn't exist
409ConflictFor example, a duplicate SKU or email
429Too Many RequestsRate limit exceeded
500Internal Server ErrorSomething failed on La Poche's side — safe to retry

Error response shape

{
  "error": {
    "code": "sku_conflict",
    "message": "A product with SKU 'JHB-001' already exists.",
    "field": "sku"
  }
}

Rate limits

Standard keys are limited to 120 requests per minute. The current limit and remaining quota are returned on every response:

X-RateLimit-Limit: 120
X-RateLimit-Remaining: 97
X-RateLimit-Reset: 1750425600

Exceeding the limit returns 429. Back off and retry after the time given in X-RateLimit-Reset.

RESOURCE 01

Products

Create, update, and list products in the catalog. Changes made here queue a sync to every connected marketplace, the same as editing a product in the dashboard.

4.1GET/products

List products

Overview

List products in the catalog, paginated.

HTTP Method
GET
Endpoint
/products
Request parameters (query string)
ParameterTypeDescription
statusstringFilter by active or inactive

limitintegerResults per page, default 25, max 100

curl https://api.lapoche.com/v2/products?status=active \
  -H "Authorization: Bearer lp_live_..."

data[].skustringStock keeping unit

data[].statusstringactive or inactive

RESOURCE 08

Webhooks

Subscribe to events instead of polling. Configure endpoints under Settings → Webhooks in the dashboard.

Available events

EventFires when
order.createdA new order is imported from any connected channel
order.status_changedAn order moves to a new status in the fulfillment workflow
inventory.low_stockA product's stock crosses its reorder threshold
marketplace.sync_failedA scheduled sync to a connected channel fails
settlement.receivedA new settlement record is received from a marketplace

Payload example

{
  "event": "order.status_changed",
  "data": {
    "order_id": "ord_2f91ac",
    "previous_status": "processing",
    "new_status": "packed"
  },
  "sent_at": "2026-06-20T08:41:00Z"
}
TIP

Verify the X-LaPoche-Signature header on every webhook request against your endpoint's signing secret before processing the payload.
